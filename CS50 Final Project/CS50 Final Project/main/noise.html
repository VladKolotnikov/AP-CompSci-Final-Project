1D Perlin Noise

As a warm up the Nature of Code book goes over Perlin Noise. Ken Perlin is a professor who was awarded the noble prize for this procedure, which is capable of generating smooth random values. He desgined this noise for quick, easy, and cheap texture generation for old Tron movie. At it's core, the function references all values that came before it to select a new random value. Furthermore it does so over multiple octaves, and then adds them up to increase detail. In this example, I just used p5 to graph the noise function.

2D Perlin Noise

As much fun as 1D noise is, arguably the most useful form of perlin noise is 2D noise. While it still has the same process of generating over octaves, its mechanic is slightly different. 2D noise covers the first row of pixels with it's 1D variant, and then moves onto the row below. From there, instead of just referencing the values that came before, it looks all around it, and roughly averages out to make a new value. It repeats this pixel row by pixel row creating all sorts of various textures. Depening on the offset of the noise, which determines smoothness, as well as its detailing(how much octaves there are) it can generate wood, marble, clouds, or any other material which has a smooth pattern to it

Random Terrain Generator

The last thing I did, which was outside the scope of the book, before embarking on the physics engine journey was create a 3d Terrain Random Generator. Now, a logical assumption here would be that I would require 3D noise (which by the way does exist, even with the p5 library), however that is not the case. 3D noise would be used for somethinglike filling a room with gas, or making a 3d cloud. As we all know however, the Earth is flat, so I think I can get away with using 2D noise by just rotating the plane. To begine, I created a mesh made up of what is called Triangle Strip. It is one way to cover a surface with mesh, and have a healthy degree of flexibility for which verteces you can pull out. Then I rotated the plan slightly, and instead of recolloring pixels like befre, I would pull out or push in verteces of the mesh in accordance to 2D noise. The result: a completely random, and infinite terrain.
